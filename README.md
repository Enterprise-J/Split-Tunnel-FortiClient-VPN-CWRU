# CWRU macOS VPN Split-Tunneler

A high-performance shell utility for macOS designed to manage FortiClient VPN connections with intelligent split-tunneling. This script ensures that only traffic destined for Case Western Reserve University (CWRU) resources (`129.22.x.x`) travels through the VPN tunnel, keeping your personal browsing, downloads, and streaming at full ISP speeds.

**Last updated: February 13, 2026**
## Features

- **Intelligent Split-Tunneling**: Automatically routes `129.22.0.0/16` through the `utun` interface while keeping the default gateway on your physical interface.
- **Mode Hot-Swapping**: Switch between Split and Full tunnel modes on the fly without disconnecting.
- **Menu Bar Indicator**: A lightweight Swift-based helper displays a Moon emoji (🌕 for Full, 🌓 for Split) in the macOS status bar to indicate your current connection state.
- **Stay-Awake Logic**: Automatically invokes `caffeinate` to prevent the VPN from dropping during system sleep (crucial for the iPad version of FortiClient on macOS).
- **Custom Route Injection**: Easily add specific IP addresses or hostnames to the VPN tunnel.
- **Clean Teardown**: The `dvpn` command ensures all routing tables are restored, DNS caches are flushed, and background helper processes are terminated.

## Prerequisites

- **macOS**: This script utilizes macOS-specific APIs (AppleScript, `scutil`, `networksetup`) and the Swift compiler (`swiftc`).
- **FortiClient VPN**: Optimized for the iPad version of FortiClient running on macOS, but adaptable for the desktop version or `openfortivpn`.
- **Sudo Privileges**: Required for modifying the system routing table. *[Enabling TouchID authentication for sudo](https://dev.to/siddhantkcode/enable-touch-id-authentication-for-sudo-on-macos-sonoma-14x-4d28) is recommended.*

## Usage

### 1. Start or Update VPN

To start a split-tunnel session (CWRU traffic only):
```bash
vpn
```

To route **all** traffic through the VPN (Full Tunnel):
```bash
vpn -a
```

### 2. Add Specific Routes

You can pass hostnames or IPs as arguments to route them specifically through the VPN:
```bash
vpn custom.server.cwru.edu 10.3.0.45
```

### 3. Check Tunneled Routes

To see exactly which destinations are currently being routed through the VPN interface:
```bash
svpn
```
### 4. Disconnect and Cleanup

To stop the VPN, kill background monitors, and restore your default routing table:
```bash
dvpn
```

## Technical Architecture

- **Swift Helper**: Upon first run, the script compiles a small Swift application (`/tmp/VPNStatusApp`) that sits in the macOS menu bar. This provides visual feedback on the tunnel state.
- **Routing Logic**: The script manipulates the routing table by breaking the default route into two halves (`0.0.0.0/1` and `128.0.0.0/1`). This allows the script to override the VPN's attempt to seize the primary default gateway.
- **Monitor Loop**: A background process monitors the `utun` interface. If the VPN disconnects, it triggers a macOS system alert and runs a cleanup routine to prevent "zombie" routes.
- **Keep-Alive**: A hidden `sudo` keep-alive loop runs in the background so you aren't prompted for a password repeatedly during long sessions or mode swaps.

## Important Notes

### Privacy
> **Warning**: This script does not prevent DNS leaking. Even in split-tunnel mode, macOS may still send DNS queries to CWRU's DNS servers.

- CWRU IT may be able to see the domains you are resolving (e.g., `google.com`, `netflix.com`), even if the actual data traffic goes through your personal ISP.
- Encrypted DNS (DoH/DoT) or iCloud Private Relay may not fully mitigate this while the tunnel is active.
- **Recommendation**: Use a private device for sensitive personal activity or disable the VPN entirely when not accessing CWRU resources.

### Compatibility
This script was specifically designed for the iPad version of FortiClient running on Apple Silicon Macs. If using the native macOS desktop client, you may need to adjust the `osascript` application ID in the `dvpn` and `vpn` functions.
